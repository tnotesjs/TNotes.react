## 3. 🤔 React 是什么时候诞生的？

- React 由 Facebook（现 Meta）于 2011 年首次开发
- 2013 年 5 月正式开源发布
- 最初是为了构建 Facebook 的新闻 feed 功能而创建

---

React 最初并不是为开源而生的，而是 Facebook（现 Meta）内部为解决实际问题而开发的工具。

2011 年，Facebook 的工程师 Jordan Walke（当时是 Facebook 的软件工程师）在开发内部项目时，遇到了一个棘手的问题：如何高效地更新复杂的用户界面？当时的主流做法是直接操作 DOM（文档对象模型），但随着应用越来越复杂，手动更新 DOM 变得繁琐、容易出错，且性能不佳。

于是，Jordan Walke 提出了一个大胆的想法：用 JavaScript 创建一个“虚拟 DOM”（Virtual DOM），在内存中模拟真实的 DOM 结构。当数据变化时，React 会先在虚拟 DOM 中计算出最小的更新差异，再一次性应用到真实 DOM 上。这种方式大大提升了性能和开发效率。

这个内部项目最初叫 “FaxJS”，后来被重命名为 React——因为它能“对数据变化做出反应（React）”。

2013 年 5 月，Facebook 在 JSConf US 大会上正式开源了 React。虽然一开始很多人对它的“单向数据流”和“JSX 语法”感到困惑甚至质疑，但 React 的简洁性和高效性很快吸引了开发者社区的关注。

## 4. 🤔 React 的重要发展节点有哪些？

### 4.1. 早期发展（2013-2015）

快速成长与社区爆发

- 2013 年：开源发布，社区开始关注
- 2014 年：发布 React Native，扩展到移动端开发
- 2015 年：发布 React 0.14，分离 ReactDOM 包

---

React 开源后迅速走红，原因有三：

1. 组件化思想：React 鼓励将 UI 拆分成独立、可复用的“组件”，比如一个按钮、一个导航栏、一个用户卡片。这种模块化方式让代码更清晰、更易维护。
2. 单向数据流：数据从父组件流向子组件，逻辑清晰，调试更容易。
3. 强大的生态系统：社区迅速围绕 React 构建了大量工具，比如 React Router（处理页面跳转）、Redux（状态管理）等。

2015 年，Facebook 推出了 React Native，允许开发者用 React 的语法开发原生移动应用（iOS 和 Android）。这标志着 React 从“网页框架”迈向“跨平台开发利器”。

### 4.2. 重要版本更新（2016-2020）

成熟与革新

- 2016 年：发布 React 15，改进服务器端渲染
- 2017 年：发布 React 16（Fiber 重构），引入错误边界、Portal 等新特性
- 2018 年：发布 React 16.3，引入新的 Context API 和生命周期方法
- 2019 年：发布 React 16.8，引入 Hooks，改变函数组件的开发方式

---

随着 React 被越来越多大公司采用（如 Netflix、Airbnb、Twitter），Facebook 团队也在不断优化它：

- 2017 年：React 16 发布，引入了全新的核心架构 Fiber，让 React 能更好地处理复杂动画、优先级调度等任务，为后续的并发特性打下基础。
- 2018 年：Hooks 横空出世！这是 React 历史上最重要的更新之一。Hooks 允许函数组件也能拥有状态（state）和生命周期等功能，大大简化了代码，减少了对类组件的依赖。从此，函数组件成为主流写法。
- 2019 年：React 团队开始推广 并发模式（Concurrent Mode） 的概念，让应用能更智能地响应用户交互（比如在加载数据时仍能响应点击）。

### 4.3. 现代 React（2020 至今）

现代 React 与未来方向

- 2020 年：发布 React 17，作为过渡版本，为未来大版本更新做准备
- 2022 年：发布 React 18，引入并发渲染、自动批处理等新特性
- 2024 年：持续更新，不断完善并发特性和支持

---

进入 2020 年代，React 更加注重开发者体验和性能优化：

- 2022 年：React 18 发布，正式支持 自动批处理（Automatic Batching） 和 并发渲染（Concurrent Rendering），让应用更流畅。
- 2023–2025 年：React 团队持续优化 React Server Components（RSC） 和 Next.js 等框架的集成，推动“服务端渲染 + 客户端交互”的混合架构，提升首屏加载速度和 SEO 表现。

如今，React 不仅是一个库，更是一种开发哲学：用组件构建用户界面，用状态驱动 UI 更新，用声明式代码描述“应该是什么样子”，而不是“如何一步步变成那样”。

## 5. 🤔 为什么 React 能成为主流？

- 组件化思想：提高代码复用性和维护性
- 虚拟 DOM：提升性能和开发体验
- 单向数据流：使应用状态更可预测
- 丰富的生态系统：大量第三方库和工具支持
- 持续创新：不断引入新特性保持竞争力
- 大厂支持：Facebook/Meta 的持续投入和维护

## 6. 🤔 为什么 React 适合初学者？

虽然 React 背后有很多高级概念，但它对新手非常友好：

- 上手门槛低：你只需要掌握基础的 HTML、CSS 和 JavaScript，就能开始写 React 组件。
- 文档优秀：React 官方文档清晰、示例丰富，还有交互式教程。
- 社区强大：遇到问题几乎都能在网上找到答案，大量开源项目可供学习参考。
- 就业前景好：React 是目前最主流的前端技术之一，掌握它能为你打开很多职业大门。

## 7. 🤔 如何跟踪 React 版本更新？

- 查阅 GitHub Releases：在 [React 的 GitHub releases][3] 页面可以查看所有历史版本的详细更新信息
- 重大更新追踪：每个大版本迭代的重要功能和变更都会在 release notes 中详细说明
- 迁移指南：版本更新时的废弃功能和迁移建议都能在发布说明中找到
- 问题排查：当遇到版本兼容性问题时，可以查询对应版本的更新日志来定位问题

## 8. 🤔 “版本兼容性问题”是什么？

- “教程”代码不工作：可能是使用的 API 在新版本中已被废弃或更改
- 功能缺失：某些功能可能在当前版本中已被移除
- 语法变更：新版本可能引入了不同的使用方式或最佳实践
- 解决方法：通过查阅 releases 页面的历史记录来了解变化详情

---

- 比如你正在看某个 react 教程，按照教程中的写法写完了程序，明明代码都一样，但是你的程序却无法正常运行，这时候很有可能是版本不同导致的错误。
- 你可以优先排查是不是某些 react 的核心 API 在当前的最新版中已经被移除了，或者是不是调用方式变了，等等。

::: warning ⚠️ 注意：“教程”的滞后性

- 某些教程可能因版本迭代而过时，导致代码无法运行。
- 比如某些书中的程序运行不了，往往是因为某些 API 在新版本中已被弃用。
- 建议：在阅读指定教程时，最好跟教程中的版本保持一致。
- 教程中的版本：
  - 可以根据教程发布时间并结合 [react/releases][3] 来推。
  - 如果可以获取到源码，直接查阅 `package.json` 中的依赖版本。

:::
